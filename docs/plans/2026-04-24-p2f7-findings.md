## Critical（必须在 P2F-7 内修复）

### C1 — `QuantScheme.__post_init__` 未验证 `granularity` 字段类型

**位置**: `src/scheme/quant_scheme.py:42-62`

`__post_init__` 对 `format` / `transform` / `round_mode` 都做了类型或值验证，**唯独 `granularity` 字段完全没验**。

```python
# 当前代码实际表现：
QuantScheme(format="fp8_e4m3", granularity="per_tensor")     # 通过（字符串当对象）
QuantScheme(format="fp8_e4m3", granularity=None)             # 通过
QuantScheme(format="fp8_e4m3", granularity=123)              # 通过
# 直到首次访问 scheme.granularity.mode 才抛 AttributeError
```

**违反**：CLAUDE.md §5.4 "`__post_init__` 验证完整性 — 必须对**全部字段**做类型验证"。

**修复**：在 `QuantScheme.__post_init__` 的 `round_mode` 验证之前（即 C2 验证之后、M1 validator 之前），增加：

```python
if not isinstance(self.granularity, GranularitySpec):
    raise TypeError(
        f"granularity must be GranularitySpec, got {type(self.granularity).__name__}"
    )
```

**配套测试**（`src/tests/test_formats_equiv.py` 追加）：

```python

    with pytest.raises(TypeError, match="granularity must be GranularitySpec"):
        QuantScheme(format="fp8_e4m3", granularity="per_tensor")

def test_quant_scheme_invalid_granularity_none_raises():
    with pytest.raises(TypeError, match="granularity must be GranularitySpec"):

    with pytest.raises(TypeError, match="granularity must be GranularitySpec"):
        QuantScheme(format="fp8_e4m3", granularity=123)
```

### C2 — `channel_axis` 负数索引：文档声称支持但构造时不做规范化或验证

**位置**: `src/scheme/granularity.py:25` docstring + `src/formats/base.py:114-131`

ADR-001 §"GranularitySpec" 声称"channel_axis 支持 NumPy 风格负索引（-1 = 最后一维）"。`GranularitySpec.__post_init__` **完全不检查负值**：

```python
GranularitySpec.per_channel(axis=-1)     # 通过，合法
GranularitySpec.per_channel(axis=-100)   # 也通过，但在 _quantize_per_channel 中：
  axis = x.ndim + axis  # 对于 ndim=4 => axis = -96，负索引传给 torch.amax(dim=-96)
  # torch 会抛 IndexError，但错误信息不指向 GranularitySpec 构造点
```

**违反**：CLAUDE.md §5.4 "维度索引的负值 — 支持负值则加文档说明；不支持则加 `axis >= 0` 校验"。当前是"隐式支持 + 无文档 + 无测试"。

**修复策略**：采用"在 `GranularitySpec` 不验证、在 `FormatBase._quantize_per_channel` 动态验证"的方案（因为 `GranularitySpec` 不持有张量形状）。

1. **修文档**：在 `GranularitySpec.channel_axis` docstring 中明确：
   > "支持 NumPy 风格负索引（如 `axis=-1` 表示最后一维）。由于 `GranularitySpec` 不持有张量形状，越界检查延迟到 `FormatBase.quantize()` 做。"

2. **修 `FormatBase._quantize_per_channel`**：在 `axis = x.ndim + axis` 规范化后加越界断言：

   ```python
   axis = granularity.channel_axis
   if axis < 0:
       axis = x.ndim + axis
   if not (0 <= axis < x.ndim):
       raise ValueError(
           f"channel_axis={granularity.channel_axis} out of range "
           f"for tensor with ndim={x.ndim}"
       )
   ```

3. **同步更新 ADR-001**：在 §"GranularitySpec" 加一行"越界检查在量化时动态做"。

**配套测试**（`src/tests/test_formats_equiv.py`）：

```python
def test_granularity_spec_per_channel_negative_axis_allowed():
    """GranularitySpec 接受负轴（不在构造期校验）。"""
    g = GranularitySpec.per_channel(axis=-1)
    assert g.channel_axis == -1

def test_format_quantize_per_channel_negative_axis_normalization():
    """_quantize_per_channel 把 -1 规范化为 ndim-1。"""
    import torch
    from src.formats.base import FormatBase
    fmt = FormatBase.from_str("fp8_e4m3")
    x = torch.randn(3, 4, 5)
    g_pos = GranularitySpec.per_channel(axis=2)
    g_neg = GranularitySpec.per_channel(axis=-1)
    assert torch.equal(
        fmt.quantize(x.clone(), g_pos, "nearest"),
        fmt.quantize(x.clone(), g_neg, "nearest"),
    )

def test_format_quantize_per_channel_out_of_bounds_axis_raises():
    """越界轴应在 quantize() 时显式报错。"""
    import torch
    from src.formats.base import FormatBase
    fmt = FormatBase.from_str("fp8_e4m3")
    x = torch.randn(3, 4)
    g = GranularitySpec.per_channel(axis=-100)
    with pytest.raises(ValueError, match="out of range"):
        fmt.quantize(x, g, "nearest")
```

## Major（随 P2F-7 一并修复）

### M1 — `QuantScheme.granularity` 默认 `per_tensor` 是 silent default

**位置**: `src/scheme/quant_scheme.py:38`

```python
granularity: GranularitySpec = field(default_factory=GranularitySpec.per_tensor)
```

`QuantScheme(format="fp8_e4m3")` 无显式 granularity 时静默采用 per_tensor。docstring 在 commit `64e8db5` 后已提到默认值，但那句放在类级 docstring（line 35），字段旁没有行内注解。

**违反**：CLAUDE.md §5.4 "无静默默认值 — 非显然默认必须 docstring 明确"。

**修复**：在 `granularity` 字段旁加注解行并扩展类 docstring：

```python
granularity: GranularitySpec = field(default_factory=GranularitySpec.per_tensor)
"""Default: PER_TENSOR (single shared scale for the whole tensor)."""
```

并在类 docstring 的 "Default:" 句子保留同样表述。**不需要新测试**（文档类修复）。

### M2 — `BFloat16Format.quantize` / `Float16Format.quantize` 重复验证 `round_mode`

**位置**: `src/formats/bf16_fp16.py:36-39, 68-71`

两个 override 里都写了：

```python
if round_mode not in _VALID_ROUND_MODES:
    raise ValueError(f"Invalid round_mode {round_mode!r}. Must be one of {_VALID_ROUND_MODES}")
```

但 `FormatBase.quantize()`（line 91-94）已经做过相同校验。要么 override 的 `super().quantize()` 会再校验一次（当 round_mode != "even" 或 per-tensor 以外时），要么 shortcut 路径完全跳过校验（当命中 `round_mode == "even" and PER_TENSOR`）。

**后果**：shortcut 路径若 `round_mode` 是 `"even"` 直接过，非 shortcut 路径走 `super().quantize()` 再校验。**当前 override 里的校验在 shortcut 路径上是"必要"的**（否则 shortcut 会跳过基类校验而把非法 round_mode 传进来）。

**选项 A**（推荐）：保留 override 内校验（shortcut 路径必需），但补一行注释说明"shortcut 路径独立校验，因为不走 super()"。

**选项 B**：重构 `FormatBase.quantize` 把 round_mode 校验提到前置、override 可以放心 `super().quantize()` 之前跑 shortcut。但这样要变更基类签名处理约定，不值得。

**选择**：A，加注释即可。**不需要新测试**。

## Minor（可在 P2F-7 内一并清理，不阻塞）

### m1 — `quantize(x, scheme=None)` 的容忍语义未文档化

**位置**: `src/quantize/elemwise.py` `quantize()` 函数

`scheme=None` 直接返回 `x`，既不报错也不量化。对比 `quantize_mx(A, scheme=None)` 有相同语义但函数 docstring 明确写了。`quantize()` 的 docstring 应补一行："`scheme is None` 时直接返回输入（不量化路径）。"

### m2 — `IntFormat.quantize` / `FPFormat.quantize` 是纯薄包装

审查发现若干 Format 子类的 `quantize()` 直接 `return super().quantize(...)` 没有任何特化。可精简为"不 override"。但这与 `FormatBase.quantize` 是 `@abstractmethod` 冲突——当前设计是"强制子类显式表态用默认路径还是 override"，薄包装是显式表态的成本。**保留，不改**。
