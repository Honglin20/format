# Phase 2 Fix (P2F): 三轴架构扶正

> **Goal**: 让 `src/` 量化核心完全不依赖 `MxSpecs`，改用 `QuantScheme(format: FormatBase, granularity: GranularitySpec, transform: TransformBase)` 三轴驱动。

**验收门**: `grep -r "MxSpecs\|mx_specs\|from.*specs import" src/` 无命中 + `pytest src/tests/ -x` 全绿

## 任务分解

### P2F-2: TransformBase + IdentityTransform + QuantScheme 升级

**Files:**
- Create: `src/scheme/transform.py`
- Modify: `src/scheme/quant_scheme.py` — 加 `transform` 字段, `format` 从 str → FormatBase
- Modify: `src/scheme/__init__.py` — 导出 TransformBase, IdentityTransform
- Modify: `src/tests/test_formats_equiv.py` — 更新 QuantScheme 测试

**Steps:**

1. 创建 `src/scheme/transform.py`：
   - `TransformBase(ABC)` — `forward(x)`, `inverse(x_q)`, `invertible: bool`
   - `IdentityTransform(TransformBase)` — `forward`/`inverse` 均为 identity

2. 升级 `QuantScheme`：
   - `format: str` → `format: FormatBase`（工厂方法仍接受字符串，内部调用 `FormatBase.from_str()`）
   - 新增 `transform: TransformBase = field(default_factory=IdentityTransform)`
   - `__post_init__` 中：若 format 是 str 则自动转换
   - 删除 `is_mx` 属性对 `granularity.is_mx` 的委托（保留，但改用 FormatBase 对象的属性）
   - 新增 `quantize(x)` 方法：`x → transform.forward(x) → format.quantize(x_t, granularity, round_mode) → transform.inverse(x_q)`
     - 注：此方法需等 P2F-3 的 `FormatBase.quantize()` 实现后才能工作，先写签名占位

3. 更新工厂方法签名保持语义不变：
   - `per_tensor(format: str | FormatBase, ...)` — 内部转 FormatBase
   - `per_channel(format: str | FormatBase, axis=0, ...)` — 同上
   - `mxfp(format: str | FormatBase, block_size=32, ...)` — 同上

4. 更新测试

### P2F-3: FormatBase 加 quantize() 抽象方法

**Files:**
- Modify: `src/formats/base.py` — 加 `quantize()` 抽象方法
- Modify: `src/formats/int_formats.py`, `fp_formats.py`, `bf16_fp16.py` — 子类实现

**Steps:**

1. `FormatBase.quantize(x, granularity, round_mode)` 抽象方法：
   - `PER_TENSOR`: 调用 `_quantize_elemwise_core`，无共享指数
   - `PER_CHANNEL`: 沿 `channel_axis` 计算 per-channel scale → 量化
   - `PER_BLOCK`: 调用 `_quantize_mx` 风格的 block quantization
   - 暂时在 `FormatBase` 中提供一个默认实现，分发到 `_quantize_elemwise_core`（对 elemwise 格式通用），子类按需 override

2. 各 Format 子类实现 `quantize()`：
   - `IntFormat.quantize()` — ebits=0 特化
   - `FPFormat.quantize()` — 通用浮点逻辑
   - `BFloat16Format.quantize()` — bf16 特化（直接 `.to(torch.bfloat16)` 快捷路径）
   - `Float16Format.quantize()` — fp16 特化

3. 从 `elemwise.py` 中提取量化核心逻辑到 `FormatBase.quantize()` 的默认实现中，让 `quantize_elemwise_op` 变为薄包装

### P2F-4: 替换 elemwise.py if-elif 链，消除 MxSpecs 依赖

**Files:**
- Modify: `src/quantize/elemwise.py` — 重写 `quantize_elemwise_op` 使用 QuantScheme
- Modify: `src/tests/test_elemwise_equiv.py` — 适配新 API（但保留旧 mx/ 对比）

**Steps:**

1. 新增 `quantize(x, scheme: QuantScheme) -> Tensor` 作为统一入口：
   ```
   x_t = scheme.transform.forward(x)
   x_q = scheme.format.quantize(x_t, scheme.granularity, scheme.round_mode)
   return scheme.transform.inverse(x_q)
   ```

2. `quantize_elemwise_op` 保留签名但改为内部构造 QuantScheme 再调用 `quantize()`（过渡兼容，P2F-6 时可能删除）

3. 删除 `quantize_elemwise_op` 中的 `mx_specs['bfloat']`, `mx_specs['fp']` if-elif 分发链

### P2F-5: 升级 mx_quantize / bfloat_quantize / vector，消除 MxSpecs

**Files:**
- Modify: `src/quantize/mx_quantize.py`
- Modify: `src/quantize/bfloat_quantize.py`
- Modify: `src/quantize/vector.py`

**Steps:**

1. `mx_quantize.py`:
   - `quantize_mx_op(A, mx_specs, ...)` → `quantize_mx(A, scheme: QuantScheme, ...)`
   - `block_size` 从 `scheme.granularity.block_size` 获取
   - `scale_bits`, `shared_exp_method`, `flush_fp32_subnorms` 作为显式参数（默认值从 ADR-004 推导：scale_bits=8, method="max", flush=False）
   - 保留旧签名作为薄兼容层（可选）

2. `bfloat_quantize.py`:
   - `quantize_bfloat(x, mx_specs, round_mode)` → `quantize_bfloat(x, scheme: QuantScheme)`
   - backward pass 中的 `get_backwards_mx_specs` 替换为构造反向 QuantScheme 的逻辑
   - 删除 `from src.specs.specs import ...`

3. `vector.py`:
   - `vec_quantize` 等函数改为接受 `scheme: QuantScheme`
   - `vec_use_exp2` / `vec_use_recip` 从 QuantScheme 层面移到算子参数

4. 删除 `src/specs/specs.py`（或移入兼容层，仅在测试中可用）
