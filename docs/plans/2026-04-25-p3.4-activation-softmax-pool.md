# P3.4 Design: Activation / Softmax / AdaptiveAvgPool

**Date**: 2026-04-25
**Status**: Approved

---

## Goal

Implement quantized activation, softmax, and adaptive average pooling operators
in `src/ops/`, bit-exact equivalent to `mx/activations.py`, `mx/softmax.py`,
and `mx/adaptive_avg_pooling.py`.

---

## Key Design Decisions

### 1. `inner_scheme` only (no OpQuantConfig)

Activation/softmax/pool operators have no weight or bias. All quantization is
per-step via vec_ops. OpQuantConfig adds no value — every field would be empty.
Each module accepts `inner_scheme: QuantScheme | None` as the sole config.

### 2. Backward quantization controlled by `inner_scheme_bw`

When `quantize_backprop=False`, mx sets `bfloat=0, fp=0` in backward specs,
making all vec_ops passthrough. We model this by storing `inner_scheme_bw` on
ctx: `None` when not quantizing backward, `inner_scheme` otherwise.

This also fixes a latent bug in Norm operators (P3.3) where backward always
used `inner_scheme` regardless of `quantize_backprop`.

### 3. Expand `src/ops/vec_ops.py`

Add `vec_exp`, `vec_exp2`, `vec_tanh` — needed by Sigmoid/SiLU/GELU/Softmax.
Re-export logic from `src/quantize/vector.py` but with `scheme` parameter.

## Operator Inventory

### Activations (`src/ops/activations.py`)

| Op | Forward pattern | Backward pattern |
|---|---|---|
| Sigmoid | q(x) → exp(-x) → +1 → recip | q(gy) → 1-sig → sig*(1-sig)*gy |
| Tanh | q(x) → tanh(x) | q(gy) → 1-tanh² → (1-tanh²)*gy |
| ReLU | relu(x) → q(output) | where(mask, gy, 0) → q(gi) |
| ReLU6 | relu6(x) → q(output) | where(mask, gy, 0) → q(gi) |
| LeakyReLU | q(x) → leaky_relu → q(out) | q(gy) → where(mask, gy, gy*slope) |
| SiLU | q(x) → sigmoid(x) → x*sigmoid | q(gy) → sig + y*(1-sig) → *gy |
| GELU | q(x) → approx-sigmoid → x*phi | q(gy) → phi + x*phi' → *gy |

Key details:
- ReLU/ReLU6 do NOT quantize input first (just mask); others do
- GELU has two modes: `first_order_gelu` (1.702*x) and detailed (1.5958*(x+0.044715*x³))
- Hardcoded bf16 coefficients: 1.703125, 0.044677734, 1.59375, 0.21386719
- GELU backward saves q_in or raw input depending on `quantize_backprop`

### Softmax (`src/ops/softmax.py`)

| Pass | Steps |
|---|---|
| Forward | q(x) → max → sub → exp/exp2 → sum → div |
| Backward | q(gy) → gy*out → sum → sub → mul(out,.) → optional *LN2_BF16 |

- `softmax_exp2` flag: when True, uses `vec_exp2` in forward and multiplies
  backward grad by `LN_2_BF16 = 0.69140625`
- `dim` normalized to non-negative on forward

### AdaptiveAvgPool (`src/ops/pooling.py`)

| Pass | Steps |
|---|---|
| Forward | Loop output positions → vec_reduce_mean per slice |
| Backward | Loop output positions → vec_add grad_delta to input slice |

- 2d only (matches mx)
- Uses `start_index`/`end_index` math helpers from mx
